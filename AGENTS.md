# Agent Guidelines

> This document provides context for AI coding assistants working on this codebase.

## Project Overview

A personal blog built with Next.js featuring interactive articles about web development, shaders, creative coding, and 3D graphics. The blog emphasizes rich interactive experiences through custom MDX components, WebGL visualizations via OGL, and animated interfaces.

**Live site:** https://blog.lenysauzet.com

## Tech Stack

| Category        | Technology                                                      |
| --------------- | --------------------------------------------------------------- |
| Framework       | Next.js 16 (App Router)                                         |
| UI              | React 19, Shadcn/UI, Radix UI primitives                        |
| Headless UI     | @base-ui/react (Select, Dialog, etc.)                           |
| Styling         | Tailwind CSS v4, CSS custom properties (oklch-based tokens)     |
| Icons           | @hugeicons/react + @hugeicons/core-free-icons                   |
| Animation       | motion (framer-motion successor)                                 |
| 3D/WebGL        | OGL                                                             |
| Content         | MDX via @next/mdx (native, no remote loading)                   |
| State           | Zustand                                                         |
| Package Manager | bun                                                             |

## MDX Content Format

Posts live in `content/` and use **JS export** syntax (not YAML frontmatter):

```mdx
export const metadata = {
  title: 'My Article Title',
  description: 'A short description for SEO and the post list.',
  tags: ['webgl', 'shaders'],
  date: 'YYYY-MM-DD',
};

Article content starts here…
```

The **slug** is derived from the filename — do not add a `slug` field to metadata.
Posts are loaded via dynamic `import()` in the App Router page.

## Component Conventions

- **Reusable components** go in `components/ComponentName/` with an `index.ts` barrel export
- **Page-level components** go in `app/_components/` — not reused outside the app shell
- **Shadcn UI** components live in `components/ui/` — generated by the CLI, avoid manual edits
- **MDX components** are registered in `mdx-components.tsx` and available in all posts
- Styles use **Tailwind utility classes** directly on JSX elements; use `cn()` from `@/lib/utils` for conditional class merging

## Path Aliases

All imports use the `@/` alias pointing to the project root:

```tsx
import { cn } from '@/lib/utils';
import siteConfig from '@/config/site';
import Anchor from '@/components/Anchor';
```

## Important Notes

- **App Router**: This project uses Next.js App Router, NOT Pages Router. Use `async` Server Components by default; add `'use client'` only when needed.
- **SSG**: Blog posts are statically generated via `generateStaticParams` + `dynamicParams = false`.
- **Shadcn/UI**: Run `bunx shadcn@latest add <component>` to add new UI primitives. Config in `components.json`.
- **Icons**: Use `@hugeicons/react` — `import { IconName } from '@hugeicons/react'`. Do not use lucide-react or heroicons.
- **No console.log**: Remove debug statements; use proper error handling.
- **CSS tokens**: Colors, spacing, and typography use CSS custom properties defined in `app/globals.css`. Never hardcode raw color values.
- **oklch**: The color system is built on `oklch()` with a `--base-hue` variable. Use `oklch()` for transparency and color manipulation.
